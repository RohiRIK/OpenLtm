# Internal Specs

Design and planning references for OpenLTM maintainers. These are **not** user-facing guides — for those, see [`../`](../README.md).

| Doc | What it is |
|-----|-----------|
| [PRD](PRD.md) | Product requirements — vision, personas, jobs-to-be-done, feature inventory |
| [ROADMAP](ROADMAP.md) | Phased delivery plan and current status |
| [DB-SPEC](DB-SPEC.md) | Database schema — every table, column, and index |
| [UX-SPEC](UX-SPEC.md) | User-facing surface and interaction model |

These documents describe intent and design rationale. When they disagree with the code, the code is authoritative — open an issue so the spec can be corrected.
